# EBS with ChatGPT 모델 평가 보고서

기준일: 2026-07-11

## 결론

현재 제품 구조는 학생용 답변과 교사용 검수 정보를 분리하고, `experiment`와 `truth` 모드를 교사가 전환하며, 학생별 실시간 채팅 telemetry를 교사용 대시보드로 전달한다.

다만 모델 품질 승인은 아직 완료 상태가 아니다. 2026-07-11 이전 실제 OpenAI 50턴 실행은 13/50만 통과했다. 이후 교사 승인 거짓 seed 보존, 현재 질문 우선 주제 선택, 독립 verifier seed 검사를 추가했으며 동일한 production 모델로 50턴을 다시 실행해야 한다.

## 모드와 정보 경계

| 구분 | 학생 화면 | 교사 화면 |
|---|---|---|
| `experiment` | 진실 맥락과 설정 Level의 통제된 거짓이 섞인 답변 | 정답, 학생용 거짓, 거짓인 이유, Level 근거, 생성·검수 결과 |
| `truth` | 독립 verifier를 통과한 사실 답변 | 사실 기준, verifier 결과, 모델·응답 ID, 차단 여부 |
| 검수 실패 | 정답이나 내부 사유 없이 재질문 안내 | 실패 단계, 검수 항목, 재시도 기록 |

`truth` 모드는 규칙 기반 답변으로 대체하지 않는다. OpenAI 설정이 없거나 독립 verifier가 승인하지 않으면 학생 전달을 차단한다. `experiment`도 production에서는 OpenAI generator와 별도 verifier를 사용하며, `LLM_PROVIDER=rules`는 로컬 테스트에서 명시한 경우에만 허용된다.

## 이번 보정

1. 현재 학생 질문의 명확한 주제를 최근 대화보다 우선해 이전 주제 반복을 막았다.
2. 짧고 모호한 후속 질문일 때만 최근 같은 학생의 대화 맥락을 사용한다.
3. 각 역사 case와 Level에 교사가 승인한 거짓 seed와 Level 근거를 제공한다.
4. LLM은 seed를 포함하되 현재 질문에 맞춘 자연스러운 진실 맥락과 대화형 표현을 생성한다.
5. 로컬 preflight와 독립 LLM verifier가 seed 보존, 실제 거짓, Level 적합, 진실 맥락, 정답·정정 비누출을 모두 검사한다.
6. 하나라도 실패하면 최대 3회 재생성하고, 끝까지 실패하면 학생에게 내부 답을 보내지 않는다.
7. 평가 증거에 턴별 `preflightFailures`, 모델, 응답 ID, 정답, 거짓, 근거를 남긴다.

## 최종 승인 조건

- 동일한 최종 commit SHA에서 자동 테스트와 CI가 통과해야 한다.
- 실제 OpenAI generator, verifier, LLM-as-judge가 50턴 모두 실행되어야 한다.
- 50/50 통과, 거짓 포함 100%, Level 적합 100%, 정답 누출 0%, fallback 0, 차단 0이어야 한다.
- 150개 OpenAI 응답 ID가 모두 존재하고 중복되지 않아야 한다.
- Cloudflare health가 `provider=openai`, `openaiConfigured=true`, `teacherProtected=true`여야 한다.
- 촬영용 room을 삭제하지 않고 두 교실 설정 rehearsal과 telemetry 검증을 완료해야 한다.
- 외부 GPT-5.5 xhigh 코드리뷰 승인과 release audit가 같은 최종 SHA를 가리켜야 한다.
