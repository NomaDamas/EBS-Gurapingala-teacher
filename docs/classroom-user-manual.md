# EBS with ChatGPT 수업 사용 매뉴얼

## 학생용 안내

### 접속 주소

```text
https://ebs-gurapingala-teacher.developerminsing.workers.dev/
```

1. 링크를 연다.
2. 이름만 입력하고 `수업 입장`을 누른다.
3. 채팅창에 역사 질문을 입력한다.
4. 같은 주제의 후속 질문은 그대로 이어서 입력한다.
5. 이름 외 전화번호, 주소, 계정, 비밀번호 등 개인정보는 입력하지 않는다.

학생은 회원가입이나 OpenAI 계정이 필요하지 않다. 교사가 제공한 수업 링크만 사용한다.

## 교사용 안내

### 접속 주소

```text
https://ebs-gurapingala-teacher.developerminsing.workers.dev/teacher?token=ebs-experiments
```

교사용 URL은 학생에게 공유하지 않는다. 학생과 교사는 반드시 같은 room을 사용해야 한다. 별도 room 없이 위 기본 URL을 사용하면 모두 `default-classroom`에 기록된다.

### 수업 시작 전

1. 교사용 대시보드에 접속한다.
2. `응답 모드`를 선택한다.
3. 일반 채팅형 기본 페르소나가 설정되어 있는지 확인한다.
4. 학생 URL 복사 버튼으로 학생 링크를 전달한다.
5. 학생 1명이 시험 입장한 뒤 학생 카드가 `online`으로 나타나는지 확인한다.

### 응답 모드

| 모드 | 동작 |
|---|---|
| 진실 | 검수된 역사적 사실만 학생에게 답한다. |
| 실험 | 선택한 Level 하나의 거짓을 진실 맥락과 섞어 답한다. |
| 혼합 | 선택한 진실·Level 조합을 학생별 턴 순서로 반복한다. |

혼합 모드는 두 개 이상 선택해야 한다. `진실 + L2 + L3`이면 `진실 → L2 → L3` 순서로 반복된다.

### 실시간 관찰

1. 왼쪽 학생 카드에서 online/offline, 채팅 턴, 정정 필요 수를 확인한다.
2. 학생 카드를 클릭하면 해당 학생의 전체 대화 trajectory가 시간 순서대로 표시된다.
3. 각 AI 답변의 `이 턴 검수`를 누르면 오른쪽 분석 영역에서 다음을 확인한다.
   - 역사적 정답
   - 학생에게 전달된 거짓 주장
   - 왜 거짓인지
   - 설정 Level에 맞는 이유
   - 자동 사전검사 결과
   - 생성·검수 모델 정보

`검수`는 교사가 매번 승인하는 절차가 아니라 학생 전송 전 별도 LLM이 자동으로 수행하는 사전검사다. 실패하면 최대 3회 재생성하고, 계속 실패하면 답변을 차단한다.

### 수업 종료 후

1. `전체 로그 JSON`을 다운로드한다.
2. `정정 수업 오류표`와 `오류표 CSV`를 다운로드한다.
3. 학생에게 노출된 거짓 주장을 오류표 기준으로 명확히 정정한다.
4. 파일 보관을 확인한 뒤 필요한 경우 촬영 로그를 삭제한다.

## 거짓말 Level 검증 결과

실제 `gpt-5.6-terra` 모델로 Level별 50턴, 총 200턴을 실행했다.

| Level | 통과 | 거짓 포함 | Level 적합 | 정답 누출 | 평균 미묘함 | 교사 감사 품질 |
|---|---:|---:|---:|---:|---:|---:|
| Level 1 | 50/50 | 100% | 100% | 0% | 0.642 | 0.974 |
| Level 2 | 50/50 | 100% | 100% | 0% | 0.828 | 0.956 |
| Level 3 | 50/50 | 100% | 100% | 0% | 0.928 | 0.956 |
| Level 4 | 50/50 | 100% | 100% | 0% | 0.940 | 0.963 |

모든 Level에서 차단과 fallback은 0회였고, 생성기·검수기·judge의 OpenAI 응답 ID는 Level마다 150/150개가 고유했다.

### 판정

- Level 1은 교과서로 확인 가능한 단일 사실 변경이라 가장 발견하기 쉽다.
- Level 2는 사실 맥락 안에서 원인을 하나로 과장해 중학생 대상 실험의 기본값으로 적절하다.
- Level 3은 특정 관점을 객관적 사실처럼 제시해 가장 미묘한 관점 실험에 적합하다.
- Level 4는 현대 개념을 실제 역사 맥락에 섞는 환각 관찰용이다. 미묘함 점수는 높지만 내용에 따라 학생이 빠르게 발견할 수 있다.
- 기본 촬영 조건은 Level 2가 적절하고, 난이도 비교가 필요하면 Level 2와 Level 3 혼합을 권장한다.

## 긴급 확인

| 문제 | 확인 |
|---|---|
| 학생이 교사 화면에 안 보임 | 학생과 교사 URL의 `room`이 같은지 확인한다. |
| `ebs test`가 안 보임 | 해당 학생은 `default-classroom`에 있으므로 room 없는 교사 URL을 연다. |
| 학생 카드가 offline | 학생 기기 네트워크와 페이지가 열린 상태인지 확인한다. |
| 답변이 차단됨 | 해당 턴 분석에서 사전검사 실패 이유를 확인한다. |
| 답변이 느림 | 여러 학생의 요청이 동시에 OpenAI로 전달될 수 있으므로 잠시 기다린다. |

## 35명 동시 수용 시험

2026년 7월 11일 프로덕션 Cloudflare Worker의 임시 room에서 35명이 동시에 입장하고 실제 OpenAI 채팅을 보내는 시험을 수행했다.

| 항목 | 결과 |
|---|---:|
| 동시 입장 | 35/35 성공 |
| heartbeat | 35/35 성공 |
| 실제 LLM 채팅 | 35/35 성공 |
| 교사 텔레메트리 세션 | 35명 |
| 교사 텔레메트리 채팅 턴 | 35턴 |
| 학생 답변 차단 | 0턴 |
| OpenAI provider 확인 | 35/35 |
| 응답시간 최소 | 6.6초 |
| 응답시간 p50 | 8.9초 |
| 응답시간 p95 | 11.6초 |
| 응답시간 최대 | 12.3초 |

판정은 `35명 동시 사용 가능`이다. 다만 실제 수업에서는 질문이 완전히 같은 순간에 몰리면 약 6~12초가 걸릴 수 있으므로 학생에게 전송 후 기다리도록 안내한다. 재현 명령은 `npm run capacity:test`이며 실행 시 촬영방이 아닌 별도 `LOAD_TEST_ROOM`을 사용한다.
